あなたは文書分析の専門家です。以下の「テキスト」から、与えられた「チェック条件」に合致するファクト（事実）を抽出してください。
ファクトとは、何かの状態や特性を述べる事実の文です。
例えば「〜である」「〜がある」「〜を含む」などの表現を含む文が該当します。

# テキスト
{text}

# 文書構造情報
{structure_summary}

# チェック条件
以下の条件リストに基づいてファクトを抽出してください。
{conditions_list}

# 指示
1. テキストから、上記の「チェック条件」に合致するファクトを抽出してください。
2. 各ファクトは独立した1つの事実を表すように分割してください。
3. 1つのファクトが複数行にわたる場合は、それを1つのファクトとして扱ってください。
4. ファクトではない文（条件や要件、意見など）は含めないでください。
5. ファクトの階層構造や関連性が明確な場合は、親子関係を表現してください。
   - 例えば、あるファクトが別のファクトの詳細や具体例である場合は、親子関係として表現します。
6. 箇条書きや地の文からファクトを抽出する際は、そのファクトが属する章や項のタイトルの内容も含めた文章として抽出してください。
   - 例えば、「第2章 システム構成」の下にある「サーバーは3台ある」というファクトは、
     「システム構成：サーバーは3台ある」のように抽出してください。
7. 箇条書きがネストしている場合、上位の箇条書きの内容も含めて抽出してください。
   - 例えば、「・ハードウェア構成」の下の「・CPUは8コア」というファクトは、
     「ハードウェア構成：CPUは8コア」のように抽出してください。
8. 箇条書きの直前に説明がある場合、その説明も含めて抽出してください。
9. 各ファクトは、関連する「チェック条件」のIDを`condition_id`として含めてください。

# 出力形式
JSONフォーマットで出力してください。以下の構造に従ってください：

```json
[
  {{
    "id": 1,
    "text": "ファクト1のテキスト",
    "item_type": "fact",
    "parent_id": null,
    "condition_id": [条件ID1, 条件ID2, ...]
  }},
  {{
    "id": 2,
    "text": "ファクト2のテキスト（ファクト1の子ファクト）",
    "item_type": "fact",
    "parent_id": 1,
    "condition_id": [条件ID3]
  }},
  {{
    "id": 3,
    "text": "ファクト3のテキスト",
    "item_type": "fact",
    "parent_id": null,
    "condition_id": [条件ID1]
  }}
]
```

- 各ファクトには一意のIDを割り当ててください。
- `item_type` は常に "fact" としてください。
- 親ファクトがない場合は、parent_idをnullにしてください。
- 子ファクトの場合は、parent_idに親ファクトのIDを指定してください。
- textにはファクトの全文を含めてください。複数行にわたる場合も1つのテキストとして扱ってください。
- `condition_id` には、そのファクトが関連するすべての条件のIDをリストで指定してください。
