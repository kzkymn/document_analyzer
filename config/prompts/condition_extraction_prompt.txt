あなたは文書分析の専門家です。以下のテキストからチェック条件を抽出してください。
チェック条件とは、何かが満たすべき基準や要件を表す文です。
例えば「〜すべき」「〜が必要」「〜を含む必要がある」などの表現を含む文が該当します。

# テキスト
{text}

# 文書構造情報
{structure_summary}

# 指示
1. テキストからチェック条件を抽出してください。
2. 条件は、その意味が明確に伝わるように適切な粒度で抽出してください。
   - 条件が複数の関連する要素を含む場合は、必ずそれらをまとめて1つの条件として抽出してください。
   - 条件の意味を理解するために必要な文脈も含めてください。
3. 1つの条件が複数行にわたる場合は、それを1つの条件として扱ってください。
4. 抽出する条件は、検証可能な具体的な要件であるべきです。抽象的すぎる表現は避けてください。
5. 条件の階層構造や関連性が明確な場合は、親子関係を表現してください。
   - 例えば、ある条件が別の条件の詳細や具体例である場合は、親子関係として表現します。
6. 箇条書きや地の文から条件を抽出する際は、その条件が属する章や項のタイトルの内容も含めた文章として抽出してください。
   - 例えば、「第3章 セキュリティ要件」の下にある「パスワードは8文字以上であること」という条件は、
     「セキュリティ要件：パスワードは8文字以上であること」のように抽出してください。
7. 箇条書きがネストしている場合、上位の箇条書きの内容も含めて抽出してください。
   - 例えば、「・ユーザー認証」の下の「・二要素認証を使用すること」という条件は、
     「ユーザー認証：二要素認証を使用すること」のように抽出してください。
8. 箇条書きの直前に説明がある場合、その説明も含めて抽出してください。

# 出力形式
JSONフォーマットで出力してください。以下の構造に従ってください：

```json
[
  {
    "id": 1,
    "text": "条件1のテキスト",
    "parent_id": null
  },
  {
    "id": 2,
    "text": "条件2のテキスト（条件1の子条件）",
    "parent_id": 1
  },
  {
    "id": 3,
    "text": "条件3のテキスト",
    "parent_id": null
  }
]
```

- 各条件には一意のIDを割り当ててください。
- 親条件がない場合は、parent_idをnullにしてください。
- 子条件の場合は、parent_idに親条件のIDを指定してください。
- textには条件の全文を含めてください。複数行にわたる場合も1つのテキストとして扱ってください。
