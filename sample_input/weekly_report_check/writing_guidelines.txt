# プログラマ週次報告書 執筆ガイドライン

## 1. 目的

本ガイドラインは、プログラマが週次報告書を作成する際の標準的な記載事項および記載方法を定め、報告内容の質を向上させ、プロジェクト管理の効率化を図ることを目的とする。

## 2. 報告書の構成

週次報告書は、以下の項目で構成されなければならない。

1. 基本情報（氏名、所属、報告期間）
2. 今週の作業概要
3. 実施タスクの詳細
4. 発生した問題点とその対応
5. 来週の予定
6. その他特記事項

## 3. 基本情報

基本情報には、以下の項目を必ず記載すること。

1. 氏名：報告者の氏名
2. 所属：報告者の所属部署・チーム名
3. 報告期間：報告対象となる期間（例：2025/4/21～2025/4/27）

## 4. 今週の作業概要

今週の作業概要には、以下の内容を簡潔に記載すること。

1. 担当プロジェクト名
2. 実施した主要タスクの概要（3行程度）
3. 全体の進捗状況（予定に対する進捗率）

## 5. 実施タスクの詳細

実施タスクの詳細には、以下の内容を具体的に記載すること。

1. 各タスクの名称
2. 各タスクの目的
3. 実施内容（具体的な作業内容、使用技術、実装方法など）
4. 成果物（作成したコード、ドキュメント、設計書など）
5. 各タスクの進捗状況（完了、進行中、保留など）

## 6. 発生した問題点とその対応

発生した問題点とその対応には、以下の内容を具体的に記載すること。

1. 問題の内容（エラー内容、バグの症状など）
2. 問題の原因（判明している場合）
3. 対応状況（解決済み、対応中、未対応）
4. 解決方法または対応方針
5. 今後の課題

## 7. 来週の予定

来週の予定には、以下の内容を具体的に記載すること。

1. 予定しているタスク名
2. 各タスクの目標と期限
3. 想定される課題やリスク

## 8. その他特記事項

その他特記事項には、上記項目に含まれない重要な情報を記載すること。例えば、以下のような内容が該当する。

1. チーム内の情報共有事項
2. 習得した新しい技術や知見
3. 提案事項
4. 研修や勉強会の参加報告

## 9. 記載上の注意事項

1. 専門用語やプロジェクト固有の略語を使用する場合は、初出時に説明を付けること
2. 具体的な数値や指標を用いて客観的に記載すること
3. 問題点や課題は隠さず正確に報告すること
4. 他者の成果物を引用する場合は、出典を明記すること
5. 機密情報の取り扱いには十分注意し、必要に応じて伏せ字やコードネームを使用すること

## 10. 提出方法

1. 週次報告書は、毎週金曜日17時までに所定のプロジェクト管理システムにアップロードすること
2. ファイル名は「週次報告_氏名_YYYYMMDD.md」の形式とすること
3. マークダウン形式で作成し、見出しや箇条書きを適切に使用すること
